# BidWorks 投标一体化生产平台 MVP 角色与权责体系 (User Roles)

为契合建筑总承包企业精细化投标的分工体系，系统定义了十个关键角色（其中包含九个业务协作伙伴与一个IT审计伙伴），并在项目全生命期赋予他们精细的生产及审批职能。

## 1. 角色矩阵与作业目标

| 对应业务岗位 | 系统简称 | 平台内核心职能 | 本MVP为其解决的核心痛点 | 数据敏感度级 |
| :--- | :--- | :--- | :--- | :--- |
| **项目负责人**<br>(Bid Manager / PM) | **项目负责人** | 管控投标进度大盘、倒排周期计划修改、查看系统总控风险、复核重大数据变更、跟进未关闭审核意见。 | 1. 无法随时掌握各专业资料真实产出进度。<br>2. 临时变更（如面积、投标截止日）无法有效传达。<br>3. 无法看透隐蔽性商务及工期风险。 | **高** (特权) |
| **营业人员**<br>(BD / Commercial) | **营业** | 投标项目发起创建、标书上传、主数据（商务段）确认、答疑文件跟踪与主数据校正。 | 1. 过去纯靠人工抄录大本招标文件关键项，易遗漏。<br>2. 追踪业主答疑变化，需频繁通过微信或邮件手动通知全员。 | **中** |
| **设计人员**<br>(Architect / Engineer) | **设计** | 领用设计工作包、确认BIM和绿建标准、上传设计方案说明说、核对计面指标、响应主数据面积变化并复核资料。 | 1. 总图和主建指标打架，不清楚概算、施工正各执何种口径面积在作业。 | **中** |
| **概算人员**<br>(Quantity Surveyor) | **概算** | 领用概算测算清单与指标提取、接收主数据条件变更的需复核状态提醒并作调测反馈。 | 1. 因设计面积变动或合同条款变更，算完的底稿全盘重来，苦于信息传递滞后。 | **高** |
| **报价决策人**<br>(Estimating Director) | **报价** | 评估项目边界款、标段边界、合同付款扣减机制、进行敏感性评估。 | 1. 报价核心明细需多岗提供支撑，但缺乏一个让敏感文件与AI隔离的安全生产环境。 | **极高** (受限) |
| **施工技术岗**<br>(Tech Specialist) | **施工技术**| 编制施工组织计划、工期优化说明、临设设计方案，在提交资料前进行平台“一致性与旧项目残留自检”。 | 1. 频繁需要套用以往类似工况方案，人名、地名、旧项目地名称极易未脱敏残留，导致废标。<br>2. 指标不一致常遭领导退回返工。 | **低** |
| **VECD 专员**<br>(Value Engineering) | **VECD** | 领用、提交VECD降低成本提案方案及其估测，维护由于提案被业主采纳或退回引发的成本测算联动。 | 1. 案例库零散且难以直接匹配当前招标特征，优化结果状态无法联动到最终概算大表。 | **中** |
| **审核领导**<br>(Director / Approver) | **审核领导** | 对各专业提交成果进行结构化连贯意见，进行针对性的段落留白审核并指派整改状态，关闭修改单。 | 1. 线上缺少划词或结构化批注。经常需要通过PDF微信截图提出十多条反馈，且极难追踪改没改。 | **中** |
| **资料汇总员**<br>(Doc Controller) | **资料汇总** | 将完成审核并已消除所有意见的终稿归集并标记“最终版”予以冻结，组织成果输出。 | 1. 汇总标书最终章时，各岗频繁交来新版文件，导致封标前版本大乱、指标前后打架。 | **低** |
| **系统管理员**<br>(IT Auditor) | **系统管理员**| 成员角色赋权、配置敏感文件特征及控制 AI 禁用隔离路径、调阅系统日志，不侵入阅读真实项目业务卷宗。 | 1. 缺乏一键全链路审计（谁上传、谁下载、谁在何时用AI读取了非涉密文件的审计机制）。 | **系统特权** (禁止越权阅读业务) |

## 2. 协作生命周期演职图

```text
  [营业] -> 上传招标书/创设项目
    │
    ▼
  [AI + 营业确认] -> 生成主数据
    │
    ▼
  [项目负责人] -> 倒排计划并指派 [设计]/[概算]/[施工]/[VECD]
    │
    ├─► [设计] ──► 产出设计大纲
    ├─► [概算] ──► 产出工程概算底稿 (受限)
    ├─► [施工] ──► 产出施组大纲、工期 (进行自检：防止残留旧项目名)
    └─► [VECD] ──► 提出降本备选方案
    │
    ▼ 
  [审核领导] -> 逐条线上审批、反馈修改
    │
    ▼ 
  [资料汇总] -> 审核意见清零、确认多岗数据一致 -> 标记 "最终版" 冻结并归档
```
