# BidWorks 投标一体化生产平台 MVP 安全权限与全面追踪 (Permissions)

信息资产的安全隔离、防爆和审计追踪是建筑总承包内部协同平台的生命线。平台基于“用户角色-项目级关联-资料敏感度”设计了三维安全矩阵，并设置了审计日志体系。

## 1. 用户、角色与项目关联 (Three-Tier Matrix)

访问控制在以下三个维度层层过滤，从而决定最终访问结果：
1. **系统角色阶层 (Role Type)**：决定系统级大功能模块（如新建项目、底线安全配置、日志调阅）是否暴露可见。
2. **项目成员权属 (Project Scope)**：即使拥有某种角色，也必须被指派加入特定项目的“项目成员池（Membership）”，否则连该项目的首页大纲都不可查阅。
3. **文件安全降解 (Document Level - Sensitive & AI Options)**：进入项目后，特定的文件、概算大底、报价算表会被强制置为“敏感（isSensitive=true）”，非指定特许的普通项目成员，即便是同一项目，也无法看阅与下载。

## 3. 全链路安全追踪和审计日志 (Logs)

任何用户的交互和后台调用数据都会无缝地入库。日志分为以下五大类：

### 3.1 文件防漏控制日志 (File Access Log)
*   **记录时机**：文件发生上传、下载、阅览（如 PDF 在线预览）、删除、作废时。
*   **必载字段**：项目ID、任务ID、发生人账号、发生人IP、文件ID、版本号、敏感度标记、交互动作等。

### 3.2 项目主数据变更足迹日志 (Master Data Drift Log)
*   **记录时机**：项目主数据库的任一核心变量发生值变迁时。
*   **必载字段**：项目ID、修改字段英文 Key、原字串/数字、新字串/数字、修改者ID、修改人客户端、确认修改时间、强制复核受众告知状态。

### 3.3 审核指导纠正意见日志 (Review Thread Log)
*   **记录时机**：审核领导点按“增加批注意见”、“更改意见责任人”、“修正确认反馈”、“亲自关闭意见”等环节。
*   **必载字段**：意见ID、关联文件ID、历史记录、发生人、时间足迹。

### 3.4 AI服务调用与引用可审计日志 (AI Interaction Log)
*   **记录时机**：系统向 AI 大语言模型、文档解析引擎、一致性校验检索等发送每次 Token 请求时。
*   **必载字段**：调用模块分类（如招标文件解析、正文一致性分析）、操作人账号、源文件大小、消耗 Token 量、引用文件、AI输出文本片段、引用页码以及“人工是否点击确认入库/或忽略”事件。

### 3.5 敏感文件 AI 只读隔离屏障防护规则
*   当一个文件在属性页被勾选为 `isSensitive = true` 或在后台归为极其敏感报价测算大底时，系统必须在 AI 读取配置文件接口直接切断：该文件的物理存储区块对 AI 解析运行算力彻底置为“零可视”，禁止任何利用 AI 总结敏感底表文件的企图，确保核心商务利润底盘在智能时代仍具备硬件级的堡垒底色保护。
