# BidWorks 投标一体化生产平台 MVP 产品立项简报 (Product Brief)

## 1. 项目愿景与背景
在建筑总承包企业中，投标项目（如“上海青浦日资制造基地项目”）具有周期短（通常仅28天）、资料多、跨专业广（涵盖营业、设计、概算、报价策略、VECD、施工、审核等多个部门）、版本变化快和责任边界复杂的特点。
目前传统的离线沟通（各岗依赖邮件、微信、共享文件夹与Excel）面临**信息孤岛、版本混乱、信息变更无法感知、检查依赖人工而频频返工**等瓶颈。

**BidWorks** 定位为**面向建筑总承包投标全过程的内部协同生产平台**。它绝不是一个只作汇报的“领导进度看板”，也不是一个死板的“OA审批流程”，而是真正嵌入各岗位一线作业的、以数据和业务协作驱动的投标生产系统。

## 2. 核心产品原则
平台设计与开发必须坚守以下五大底线原则：
1. **先服务一线，再服务管理层**：系统必须首先帮助一线使用者省时省力（如辅助摘要、自动检查、版本对照），管理层所需的项目全局状态和风险数据应在各岗位行使自身生产职责时自然产生。
2. **只要用户上传资料，平台必须即时回报**：坚决避免“纯填报式”系统。用户每上传一个文件，系统应即时反哺版本生成、内容摘要、一致性自检、招标响应标记、变更影响提醒等有价值的闭环反馈。
3. **主数据只维护一次，多处自动引用**：项目关键指标（如项目名称、业主名称、建筑面积、总工期、投标截止日、招标范围等）作为核心信息源，严禁在不同的Word、PPT、Excel中各自重复录入或拼写不一。
4. **一处信息变化，相关环节即时感知并促成复核**：如建筑面积或工期改变，相关岗位（设计、概算、施工、报价等）必须收到联动预警与差异对比，相关资料自动标记为“待复核”状态。
5. **AI仅作辅助，不替代专业决策，结果必须有据可循**：AI解析招标文件时，必须提供精准的源文档定位（如页码、段落），不可自动代行最终审批或静默修改高风险专业资料。敏感资料默认对AI不可读。

## 3. 用户价值与商业成效
- **对一线岗位**：减少在繁杂招标资料中检索信息的时间；从上传即自检中快速纠错；利用历史及推荐结构加速方案编制。
- **对审核领导与项目负责人**：随时通过项目总控台看透项目核心风险、缺项、变更待复核节点以及未关闭审核意见，告别最终汇总时的“致命返工”。
- **对企业IT与信息安全**：建立项目、角色、文件三位一体的安全拦截，规范日志管理（包含主数据变更、文件上下传、AI调用等审计），保证核心商务敏感数据（如报价明细、测算底稿）绝对受控。
